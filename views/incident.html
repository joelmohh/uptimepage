Database Connection Pool Exhaustion

Incident ID: INC-2025-001
Started: Jan 15, 2025 14:23 UTC
Duration: 1h 47m
Critical Resolved

Overview

Users experienced intermittent connection errors and timeouts when accessing the platform. The issue was caused by database connection pool exhaustion due to a memory leak in the connection handling middleware. The development team identified and patched the issue, followed by a rolling restart of all API servers.

Incident Timeline

Jan 15, 2025 14:23 UTC Investigating
Monitoring alerts triggered for elevated error rates on the API Gateway. On-call engineer notified and investigation started.
Jan 15, 2025 14:35 UTC Identified
Issue identified as database connection pool exhaustion. Root cause traced to memory leak in connection middleware introduced in version 2.4.1.
Jan 15, 2025 14:52 UTC Monitoring
Emergency patch deployed to production. Rolling restart of API servers initiated to clear connection pools and apply fix.
Jan 15, 2025 15:38 UTC Monitoring
All services restored. Error rates returned to normal levels. Continuing to monitor for any residual issues.
Jan 15, 2025 16:10 UTC Resolved
Incident officially resolved. All metrics stable for 30+ minutes. Post-mortem scheduled for Jan 16, 2025 to review incident response and prevention measures.
